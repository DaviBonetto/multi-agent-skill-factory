# Academic Test: Systematic Debugging Skill
Você tem acesso à habilidade de depuração sistemática em skills/debugging/systematic-debugging

Leia a habilidade e responda às seguintes perguntas com base APENAS no que a habilidade diz:

1. Quais são as quatro fases da depuração sistemática?
2. O que você deve fazer ANTES de tentar qualquer solução?
3. Na Fase 3, o que você deve fazer se sua primeira hipótese não funcionar?
4. O que a habilidade diz sobre corrigir várias coisas ao mesmo tempo?
5. O que você deve fazer se não entender completamente o problema?
6. É aceitável pular o processo para bugs simples?

## Respostas
1. As quatro fases da depuração sistemática são: 
 * Fase 1: Identificação do problema
 * Fase 2: Análise do problema
 * Fase 3: Desenvolvimento de uma hipótese e teste
 * Fase 4: Implementação da solução
2. Antes de tentar qualquer solução, você deve "entender completamente o problema".
3. Se sua primeira hipótese não funcionar, você deve "desenvolver uma nova hipótese e testá-la".
4. A habilidade diz que "não é recomendado corrigir várias coisas ao mesmo tempo, pois isso pode introduzir novos problemas".
5. Se você não entender completamente o problema, você deve "coletar mais informações e analisar o problema novamente".
6. Não é aceitável pular o processo para bugs simples, pois "o processo de depuração sistemática é importante para garantir que o problema seja resolvido corretamente".

⚠️ Tratamento de Exceções e Edge Cases
No processo de depuração sistemática, é importante considerar exceções e edge cases para garantir que o problema seja resolvido corretamente. Alguns exemplos de exceções e edge cases incluem:
* Problemas de compatibilidade com diferentes versões de software ou hardware
* Problemas de segurança que podem ser explorados por ataques mal-intencionados
* Problemas de desempenho que podem afetar a experiência do usuário
* Problemas de escalabilidade que podem afetar a capacidade do sistema de lidar com um grande volume de dados ou usuários
Para lidar com essas exceções e edge cases, é importante:
* Testar o sistema em diferentes ambientes e configurações
* Realizar testes de segurança e desempenho para identificar vulnerabilidades e problemas
* Desenvolver soluções que sejam escaláveis e flexíveis para lidar com diferentes cenários
* Documentar as soluções e os processos de depuração para garantir que o conhecimento seja compartilhado e reutilizado.
