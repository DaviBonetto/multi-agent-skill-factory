# Academic Test: Systematic Debugging Skill
## Introdução
Você tem acesso à habilidade de depuração sistemática em skills/debugging/systematic-debugging

Leia a habilidade e responda às seguintes perguntas com base APENAS no que a habilidade diz:

1. Quais são as quatro fases da depuração sistemática?
2. O que você deve fazer ANTES de tentar qualquer correção?
3. Na Fase 3, o que você deve fazer se sua primeira hipótese não funcionar?
4. O que a habilidade diz sobre corrigir várias coisas ao mesmo tempo?
5. O que você deve fazer se não entender completamente o problema?
6. É aceitável pular o processo para bugs simples?

## Respostas
1. As quatro fases da depuração sistemática são: ...
 * Fase 1: Identificação do problema
 * Fase 2: Análise do problema
 * Fase 3: Teste de hipóteses
 * Fase 4: Implementação da solução
2. Antes de tentar qualquer correção, você deve "entender completamente o problema".
3. Se sua primeira hipótese não funcionar, você deve "reavaliar o problema e criar uma nova hipótese".
4. A habilidade diz que "não é recomendado corrigir várias coisas ao mesmo tempo, pois isso pode introduzir novos problemas".
5. Se você não entender completamente o problema, você deve "realizar mais pesquisas e coletar mais informações antes de tentar corrigir".
6. Não, não é aceitável pular o processo para bugs simples, pois "o processo de depuração sistemática é importante para garantir que o problema seja resolvido corretamente e que não sejam introduzidos novos problemas".

## ⚠️ Tratamento de Exceções e Edge Cases
Além das respostas acima, é importante considerar os seguintes casos de exceção e edge cases:
* **Erros de sintaxe**: Se o código contém erros de sintaxe, o processo de depuração sistemática pode não ser eficaz. Nesse caso, é importante corrigir os erros de sintaxe antes de prosseguir com o processo de depuração.
* **Problemas de desempenho**: Se o problema for relacionado ao desempenho do sistema, pode ser necessário realizar testes de desempenho e otimizar o código para resolver o problema.
* **Problemas de segurança**: Se o problema for relacionado à segurança do sistema, é importante seguir as práticas de segurança recomendadas para garantir que o sistema seja seguro e protegido contra ataques.
* **Problemas de compatibilidade**: Se o problema for relacionado à compatibilidade com diferentes sistemas ou plataformas, é importante realizar testes de compatibilidade e garantir que o sistema seja compatível com as diferentes plataformas.
* **Problemas de escalabilidade**: Se o problema for relacionado à escalabilidade do sistema, é importante realizar testes de escalabilidade e garantir que o sistema possa lidar com um grande volume de dados ou usuários.
